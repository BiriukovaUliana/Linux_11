## Обгрунтування:
Завдання вимагало створити обробник сигналу SIGBUS. Для реалізації було обрано мову C.

Замість стандартної функції signal(), було використано сучасний системний виклик sigaction(). Це дозволило встановити прапорець SA_SIGINFO.
Завдяки цьому обробник отримав доступ до структури siginfo_t, яка містить поле si_code для ідентифікації причини сигналу та si_addr для визначення адреси, де стався збій.

### В самому обробнику була реалізована перевірка кодів:
1. BUS_ADRERR: ідентифікує помилки, пов'язані з відображенням файлів/
2. BUS_OBJERR: вказує на специфічні помилки об'єкта пам'яті.

### Щоб перевірити роботу коду, була створена штучна ситуація:
- Файл було відображено в пам'ять через mmap().
- Розмір файлу було примусово змінено до нуля через ftruncate().
- Спроба звернення до вже порожнього віртуального вікна викликала сигнал SIGBUS.

В результаті програма успішно перехопила сигнал, вивела точну адресу помилки та правильно визначила її тип як BUS_ADRER
